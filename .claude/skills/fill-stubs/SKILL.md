# fill-stubs: 빈 개념 노트 채우기

DontPanic 볼트의 `Concepts/` 폴더에서 제목만 있고 내용이 없는 노트를 찾아, 웹 서치로 조사한 내용으로 채운다. **볼트 원본에 직접 쓴다** (다음 sync 주기에 위키에 자동 반영됨).

볼트 경로: `/Users/giy/Library/Mobile Documents/iCloud~md~obsidian/Documents/DontPanic`

## 절차

### 1. 스텁 탐지
`Concepts/` 폴더에서 후보 찾기:
```bash
find "$VAULT/Concepts" -name "*.md" -size -300c
```
각 후보를 Read로 확인해 스텁 여부 판정: 프론트매터를 제외한 실질 본문이 80자 미만이거나 위키링크만 있으면 스텁. 이미 `ai-generated: true`인 파일은 제외.

사용자가 특정 노트를 지정했으면 그것만, 아니면 목록을 보여주고 **한 번에 최대 5개**까지 처리 (사용자가 명시적으로 더 요청하면 초과 가능).

### 2. 조사 및 작성 (병렬 가능)
여러 개를 처리할 때는 **에이전트 최대 5개 병렬** (스텁 1개당 1에이전트, general-purpose). 각 에이전트에게:
- 개념 이름(파일명)과 기존 파일 내용(보존할 위키링크 포함) 전달
- WebSearch/WebFetch로 조사 (한국어+영어 검색 병행, 신뢰할 만한 출처: 학술 자료, 사전, 위키백과, 전문 아티클)
- 볼트 Concepts의 전체 파일 목록 전달 → **볼트에 실존하는 노트로만 위키링크** (깨진 링크 금지)

### 3. 노트 형식
이 볼트는 한국어 인문학/미디어이론 연구 볼트다. 볼트 자체 템플릿(`templates/tpl_concept.md`) 구조를 따른다:

```markdown
---
type: concept
tags: [관련, 태그]
ai-generated: true
ai-generated-date: YYYY-MM-DD
sources:
  - https://출처1
  - https://출처2
---

# 노트
(개념의 핵심을 한국어로 2-4단락. 제안자/기원, 정의, 왜 중요한지.
학술적이되 자기 말로 풀어쓴 톤. 인문학/미디어이론/예술 맥락 우선.)

## 관련내용
(핵심 논점, 주요 문헌·이론가, 개념의 확장이나 논쟁)

## 연결
(기존 파일에 있던 위키링크는 반드시 보존 + 볼트에 실존하는 관련 노트 링크 추가)

## 열린 질문
(이 개념에서 파생되는 탐구 질문 1-3개)
```

### 4. 쓰기 직전 안전 확인
파일에 쓰기 **직전에 반드시 다시 Read**해서 여전히 스텁인지 확인. 그 사이 사용자가 내용을 채웠으면 skip (iCloud 다중 기기 편집 충돌 방어). 기존 프론트매터가 있으면 키를 보존하며 병합.

### 5. 보고
처리 완료 후 사용자에게 보고: 채운 노트 목록(개념별 한 줄 요약), skip한 것과 이유, 사용한 주요 출처. AI가 채운 노트는 옵시디언에서 `ai-generated` 태그로 검색 가능함을 안내.
